## 一、主流程說明

整個系統流程從「銷售預測」出發，經過「需求預測」、「物流配送」到「財務記帳」，形成完整閉環。

```
【銷售資料來源】
漢堡王中繼系統（新零售）每日傳入門市銷售資料
  ↓ POS 端收單時標記員工餐，新零售端傳入時拆分：純銷售 / 員工餐（待確認 BK3-02）
  ↓
【需求預測】
BOM 食材需求試算（#24）
  → 依 SHORT/LONG 判斷預測方式
      SHORT（短效期）：每次都走萬元預測
      LONG（長效期） ：庫存充足則跳過，庫存不足才預測
  → 萬元用量只計算純銷售（排除損耗與員工餐）
  → 員工餐用量獨立顯示並落 DB
  → 合併臨時需求審核（#26）的加單
      ※ 同一需求日同時有預測單與臨時加單時，臨時加單食材合併顯示（BE 缺陷修復中）
  ↓
非 BOM 物料需求試算（#25）
  → 合併所有門市食材需求
  → 依廠商報價「適用門市」設定綁定正確廠商
  → 計算建議訂貨量
      公式：萬元需求 × 預估營業額 - 盤點量 - 在途量
      在途量（門市）：已向倉儲下物流單未到貨（MSS 已上傳 SFTP 但未確認收貨）
      在途量（總部）：已向供應商下採購單未入庫
  → 門市可在最終訂貨量欄位微調（上下限由系統參數控制）
  → 系統自動換算門市操作量（箱數）
  ↓
原物料需求行事曆（#30）
  → 以日曆視角呈現「日期 × 門市 × 食材」需求
  → 單號可點擊跳轉至對應單據（預測單 / 臨時需求單）
  ↓
【物流配送】
物流管理行事曆（#48）[門市收貨確認，唯一有寫入動作的頁面]
  → 藍色圓點：MSS 中央倉配送
  → 紅色圓點：廠商直送（每家廠商獨立一點）
  → 點圓點載入明細，實際配送數量預設帶入預計數量
  → 門市只需修改缺少或異常的品項
  → 按「儲存」直接建立 WHS 入庫單（當日即完成入帳）
  ↓
【天心財務串接（變更新增）】

進貨單（門市在物流管理行事曆按「儲存」建立入庫單後非同步觸發）
  → 依廠商報價 useDeliveryType 判斷配送方
      直送：配送方 = 外部廠商
      週配/月配：配送方 = 總公司（中央倉）
  → 天心進貨單數量以門市確認收貨的入庫單數量為準

【物流查詢功能（獨立進行，純查詢）】

物流規劃查詢（#49）
  → 以「預計配送日」分組，供物流規劃人員排車排路線

物流配送查詢（#50）
  → 以「預計配送日 × 門市 × 原物料」三維視角查詢配送狀況
  → 每日自動產出 CSV 上傳至 MSS 倉儲系統（SFTP）
  → 支援下載特定品項每日用量表（FE 補按鈕）

物流單查詢
  → 以「物流單」為主鍵逐單追蹤配送狀況

銷貨單 + 組合單（每日批次，待確認 T4-03 排程時間點）
  → 每門市每日兩張
      一般銷售銷貨單
      員工餐銷貨單（待確認 T4-04 品項辨別標記）
  → 每張附帶各品項 BOM 組合單

基礎主資料同步
  → 食材品號新增/停用/對應關係變更時即時觸發
  → 品號對應表（ext_prod_code_mapping）即時更新

天心財務系統
  → 依進貨單計算食材成本
  → 依銷貨單 + 組合單扣庫存、計算銷貨成本
  → 產出各門市損益表
```

## 二、角色負責項目說明

---

### 角色一：門市店長

#### 負責項目

| 項目 | 說明 |
|-----|------|
| 食材需求預測確認 | 查看系統建議訂貨量，依需要微調最終訂貨量 |
| 收貨確認與入庫 | 在物流管理行事曆確認實際到貨數量，按儲存建立入庫單，當日完成入帳 |
| 需求行事曆查詢 | 查看原物料需求行事曆，確認預測與臨時加單的食材需求完整性 |

#### 操作情境

**叫貨（非 BOM 試算明細頁）**

店長進入「需求集合 → 物料需求預測」，變更後畫面提供完整資訊：

| 欄位 | 說明 |
|-----|------|
| 庫存類型（SHORT/LONG） | 來自 PDM 食材主檔，反映食材儲存特性 |
| 建議訂貨量 | 萬元需求 × 預估營業額 - 盤點量 - 在途量，最小為 0 |
| 盤點量 | 試算當下 WHS 庫存即時餘額 |
| 在途量 | 已向倉儲（MSS）下物流單但尚未到貨的數量 |
| 門市操作量（箱） | 系統依廠商報價箱規自動換算，直接看箱數下單 |
| 最終訂貨量 | 可編輯，上下限由系統參數控制，超出時顯示提示 |
| 員工餐用量（平日/假日） | 來源為 POS 端收單時標記為員工餐的記錄，由新零售端傳入時拆分，獨立顯示，不納入預測計算 |

**SHORT 食材**（如麵包、生鮮）：每次都顯示建議訂貨量。

**LONG 食材**（如醬料、冷凍肉品）：
- 庫存充足（≥ 安全存量）→ 顯示灰色「判斷庫存」，不需補貨
- 庫存不足（< 安全存量）→ 顯示橘色警示 + 建議訂貨量

**防呆提示**

| 情況 | 標示 |
|-----|------|
| SHORT 食材計算後建議量為 0 | 黃色底色，提示「請確認是否需要訂貨」 |
| 建議量超過歷史均值 3 倍（系統預設閾值） | 紅色底色，提示「建議量異常偏高，請確認後調整」 |
| 某食材查無廠商報價 | 頁面頂部橘色警示橫幅，對應列橘色標示，廠商欄顯示「無報價」；確認後可繼續使用其他食材的試算結果 |

**多廠商對應**

若同一品號有多家廠商，採購人員只需在廠商報價維護設定「適用門市」，系統就會自動對應正確廠商與箱規，店長不需要自行判斷。店長在畫面上也不會看到廠商對應的資訊，系統已在背景自動處理。

**用量表下載（新增）**

店長或規劃人員可在物料需求預測試算頁面，點擊「下載用量表」按鈕，選擇品項範圍與查詢期間，下載特定品項的每日用量表，作為訂貨參考依據。

**原物料需求行事曆（操作改善）**

進入「原物料需求行事曆」查詢時：
- 單號欄位現在可以點擊跳轉——點擊預測單號直接跳轉至對應的 BOM 試算單據，點擊臨時需求單號直接跳轉至對應的臨時需求審核單據
- 同一需求日同時有預測單與臨時加單時，兩者的食材需求會合併顯示，不會漏掉臨時加單的品項

**收貨（物流管理行事曆）**

變更後收貨操作更簡化，且要求當日完成入帳：

1. 進入「物流管理行事曆」，系統自動帶入自己的門市，按「查詢」
2. 月曆上藍色圓點＝中央倉配送，紅色圓點＝廠商直送（店長看不到廠商名）
3. 點圓點 → 明細表格載入，**實際配送數量欄位預設帶入預計發送配送數量**
4. 店長只需修改缺少或數量不符的品項，其餘維持預設值
5. 按「儲存」建立入庫單，系統同步推送天心財務進貨單

> 注意一：進貨應在當日完成確認，確保入庫單日期等於實際進貨日期，避免月/週盤時出現盤點差異。
> 注意二：入庫單建立後欄位即鎖定，若數量有誤需至庫存模組處理，同時會影響天心的財務記帳數字，請確認後再按儲存。

### 角色二：廠商配送人員

#### 負責項目

| 項目 | 說明 |
|-----|------|
| 直送到門市 | 依採購單將貨品直接配送至指定門市 |
| 品號對應確認 | 確認送貨品項與力新 ERP 品號一致 |

#### 操作情境

廠商配送人員本身不直接操作力新 ERP，但其配送行為在系統中的記錄方式如下：

**收貨確認端（由門市操作）**

廠商送達門市後，由門市店長在物流管理行事曆點選對應廠商的紅色圓點，確認實際到貨數量後按儲存建立入庫單。

**天心串接連動（新增）**

入庫單建立後，系統依廠商報價的 `useDeliveryType` 判斷此筆為「廠商直送」，自動推送天心進貨單，配送方填入「該廠商」。

**廠商需配合的事項**

| 事項 | 說明 |
|-----|------|
| 品號一致性 | 送貨品項的品號需與廠商報價維護中登記的品號一致 |
| 新增或停用品項 | 需提前通知採購，由採購人員更新廠商報價維護。系統會即時同步更新品號對應表並觸發天心基礎主資料同步 |

### 角色三：總部（採購 / 物流規劃 / 供應鏈）

#### 負責項目

| 項目 | 說明 |
|-----|------|
| 食材需求規劃 | 執行 BOM 試算與非 BOM 試算，掌握整體食材需求 |
| 廠商報價維護 | 設定廠商報價、適用門市、物流類型 |
| 食材主檔維護 | 設定食材庫存類型（SHORT/LONG），必填欄位 |
| 採購流程執行 | 請購 → 採購報價 → 採購管理 → 驗收確認 |
| 物流規劃 | 掌握整月配送負擔，排車排路線 |
| 品號對應表維護 | 維護力新 ↔ 天心品號對應表（新增） |

#### 操作情境

**食材主檔維護（新增必填欄位）**

食材維護作業新增「庫存類型」必填欄位（SHORT / LONG），建立或編輯食材時必須設定，此設定直接影響需求預測的計算方式。

**食材需求預測試算（BOM #24）**

試算結果新增顯示：

| 新增欄位 | 說明 |
|---------|------|
| 庫存類型（SHORT/LONG） | 來自食材主檔，在試算明細頁直接顯示 |
| 員工餐用量（平日/假日） | 來源為 POS 端收單標記，由新零售端傳入時拆分，獨立顯示，不納入萬元計算（待確認 BK3-02） |
| 「判斷庫存」標示 | LONG 食材庫存充足時顯示，代表本次跳過萬元計算 |
| 安全存量警示 | LONG 食材庫存不足時顯示橘色警示與當前庫存數值 |

> 注意：萬元用量計算基礎已改為純銷售數量，排除損耗與員工餐，計算結果會比以往低，屬於正常。

**廠商報價維護（多廠商對應）**

當同一品號有多家廠商供貨時，只需在廠商報價維護為每家廠商設定正確的「適用門市」，系統即可自動對應，不需修改系統其他設定。

> 注意：若同一品號有不同單價的多家廠商，會影響天心端的 PLV 損益計算準確度。均值成本計算規則正在與天心確認中（待確認 T6-01）。

**採購驗收流程（天心串接新增）**

驗收確認歸檔後，系統新增非同步推送天心進貨單的步驟，對採購人員透明，不影響現有操作。判斷邏輯：

| 廠商報價 `useDeliveryType` 對應 `pdm_logistics_type` | 天心進貨單配送方 |
|--------------------------------------------------|---------------|
| 直送 | 外部廠商 |
| 週配 / 月配（中央倉） | 總公司（中央倉統編） |

**品號對應表維護（新增）**

系統新增「力新 ↔ 天心品號對應表（`ext_prod_code_mapping`）」：
- 食材品號新增時：自動建立記錄，天心代號預設與力新品號相同
- 食材品號停用時：自動同步通知天心
- 品號對應關係變更時：`ext_prod_code_mapping` 即時更新並觸發天心同步
- 若兩端品號不同：由 IT 或採購人員人工覆寫 `tian_xin_prod_code` 欄位

**月度物流規劃（物流管理行事曆）**

總部物流規劃人員以月度視角掌握整區配送負擔分布，月曆上可看到廠商名首字（非店長層級，完整資訊可見），協助排車排路線。

### 角色四：財務人員

#### 負責項目

| 項目 | 說明 |
|-----|------|
| 門市損益表查詢 | 透過天心財務系統查看各門市損益 |
| 資料異常確認 | 當損益表資料缺漏或異常時，與 IT 確認串接記錄 |

#### 操作情境

財務人員透過**力新 ERP 系統**查詢門市損益表。力新 ERP 在背後與天心財務系統串接，由天心負責成本計算後將結果回傳，財務人員不需要直接操作天心系統。

**三條資料流**

| 資料流 | 觸發時機 | 內容 | 天心用途 |
|-------|---------|------|---------|
| 基礎主資料 | 食材新增/停用/品號變更時即時觸發 | 品號 + 門市部門 | 建立貨品主檔 |
| 進貨單 | 驗收確認歸檔後（非同步） | 品號 + 數量 + 單價 + 配送方 | 計算食材進貨成本 |
| 銷貨單 + 組合單 | 每日批次（時間點待確認 T4-03） | 各門市各品項銷售數量 + BOM 食材耗用量 | 扣庫存 + 計算銷貨成本 |

**損益表結構說明**

- 各門市視為獨立法人（獨立統編、獨立損益表）
- 天心以集團模式 + 部門分類實現總部可查看各門市損益彙總
- 員工餐成本來源為 POS 端收單時標記為員工餐的銷售記錄，與一般銷售成本以不同傳票模板區分會計科目
- 中央倉配送到門市：進貨單配送方為「總公司」
- 廠商直送到門市：進貨單配送方為「外部廠商」

**財務人員需要注意的事項**

| 事項 | 說明 |
|-----|------|
| 套餐金額拆分規則（待確認 T4-01） | 套餐銷售金額如何拆分到各子品項，由 PM + BK 業務確認中，確認前套餐毛利數字可能不準確 |
| 每日推送時間點（待確認 T4-03） | 由天心 + PM 確認排程時間點，確認後財務人員可知道損益表資料的更新時效 |
| 多廠商均值成本（待確認 T6-01） | 同一品號有多家廠商時，天心端均值成本計算規則待確認，確認前多廠商品項的損益數字可能有差異 |
| 品號轉換成本差異（待確認 T2-02） | 品號變更後力新端即時同步天心，但天心端成本計算週期是否即時反映需與天心確認 |
| 資料缺漏的處理方式 | 若發現損益表某門市某日資料缺漏，請通知 IT 確認天心串接記錄作業，IT 可從系統補發推送 |

## 三、角色影響總覽

| 角色 | 主要影響 | 新增操作 | 注意事項 |
|-----|---------|---------|---------|
| **門市店長** | 叫貨畫面有建議訂貨量、防呆提示、箱數換算；收貨只需改缺少品項；行事曆單號可跳轉；臨時加單食材完整顯示 | 查看 SHORT/LONG 標示；確認無廠商警示；微調最終訂貨量；當日完成收貨確認 | 入庫後連動財務，確認數量再按儲存；進貨需當日入帳 |
| **廠商配送人員** | 直送入庫後自動推送天心財務，品號需一致 | 無系統操作 | 新增/停用品項需提前通知採購 |
| **總部採購/物流** | 需求預測新增員工餐欄位與庫存類型；採購驗收後新增天心推送；收貨操作簡化 | 設定食材主檔庫存類型（必填）；維護廠商報價適用門市；維護品號對應表 | 萬元用量計算基礎改為純銷售，數字會低於以往；多廠商均值成本待確認 |
| **財務人員** | 可在力新 ERP 查門市損益表（天心在背後計算） | 了解三條資料流；異常時通知 IT 確認串接記錄 | 套餐金額拆分規則確認前套餐毛利可能不準；多廠商均值成本待確認；品號轉換成本差異待確認 |
