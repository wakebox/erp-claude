## 1. 功能概覽

### 1.1 我是誰

我是漢堡王台灣 **某分店店長**（例：BK001 信義店）。

我每天開店前、每週排班前，需要一眼看到「**我的店這一週／這個月，哪幾天會收到什麼**」。
別人看的是「全公司物流負擔」，我看的是「**我的店的日曆**」 — 哪天到貨多、要備幾名點收人手、哪天可以放心安排活動。

### 1.2 我要做什麼

- 每日早班開店前：打開行事曆，看當天是否有配送
- 每週日晚上：看下週每天的配送量，預排班表
- 行銷活動前：確認大型物資（紙袋、贈品包材）的預計到貨日
- 異常追蹤：發現某天「該到卻沒到」，跳到其他物流功能查原因

### 1.3 我有什麼需求

| 我的需求 | 背後的問題 |
|---|---|
| 用日曆方式看下週／下個月的到貨節奏 | 列表式看不出哪天忙、哪天閒 |
| 同一天分開呈現週配與月配 | 兩種配送車不同，點收時段也不同 |
| 點某一天可看當日詳細品項 | 排班需要知道體積與重量級距 |
| 只看自己店 | 全公司資料對店長無意義 |
| 行動裝置上看 | 賣場現場、休假在家都需要看 |

### 1.4 因此，需要以下功能

| 功能 | 解決的問題 |
|---|---|
| 行事曆視圖（日／週／月切換） | 直覺看到本月節奏 |
| 同日多配送模式分區呈現 | 區分早班週配、午班月配 |
| 點某日 → 跳出當日配送清單 | 看品項與數量 |
| 異常標記（已到／待到／延誤） | 一眼看到異常日 |
| 與其他物流功能相互跳轉 | 異常時直接進細節頁 |

### 1.5 功能基本資訊

| 項目 | 內容 |
|---|---|
| 功能中文名 | 物流管理行事曆（店長視角） |
| 所屬模組 | Excel 列「店長功能」；底層實作在「需求集合（PDM）」 |
| 兄弟功能 | 物流規劃查詢、物流配送查詢、物流異動查詢、物流串接記錄 |
| 主要頁面 | 月曆／週曆主頁、單日明細彈窗 |
| 簽核流程 | 無（純查詢） |
| 後端店長專屬 endpoint | **無**（推論：未實作；現用總部共用 API） |

## 2. 功能目的

讓店長能用「**日曆**」這種最符合人腦排程習慣的方式，看見「**自己這家店**」在未來一段時間內的物流節奏。
與兄弟功能相比，行事曆是「**入口型儀表板**」 — 不負責顯示明細，負責讓店長**找到該關心的那一天**，再跳到其他功能看細節。

設計理念：
- **以時間為主軸**（不是以單號、不是以食材）
- **預先看見**（看未來，不是事後查），與「物流規劃查詢」「物流配送查詢」「物流異動查詢」事後追蹤性質互補
- **唯讀＋鎖店**

## 3. 業務邏輯背景

### 3.1 行事曆的資料來源

行事曆顯示的不是「物流公司已派車」的資料，而是「**需求預測與臨時需求所推算出的『預計配送日』**」。

也就是說，**店長在行事曆上看到的某一天**，背後是這條鏈：

```
店長之前送出的需求預測  →  系統算出預計配送日  →  進到行事曆某一格
```

> 這代表：行事曆只能呈現「**已被需求預測涵蓋**」的配送。臨時加單若還沒被簽核完，可能不會顯示（待確認）。

### 3.2 同一天可能有多種配送模式

底層資料把配送分為三種模式（週配／月配／直送），常見情況是：

- 一個月配日（每月固定 1～2 天大量月配進來）
- 多個週配日（一週 2～3 次補日常用品）
- 偶發直送日（特殊食材直接由供應商送）

行事曆同一格內，應**分區呈現這三類**，避免店長看到一團糟。

### 3.3 與兄弟物流功能的差異（店長視角）

| 功能 | 看什麼 | 主要時機 |
|---|---|---|
| **物流管理行事曆（本功能）** | **未來日曆視圖、每天概況** | **排班、預期備料** |
| 物流規劃查詢 | 每天會送哪些品項、來自哪張預測單 | 排班前一天的細節查詢 |
| 物流配送查詢 | 已派車、現在配送中的狀態 | 收貨當下 |
| 物流異動查詢 | 實際到店 vs 預計的差異、延誤、短裝 | 收貨後對帳 |
| 物流串接記錄 | ERP 向中央倉送出的批次紀錄 | 每日對帳、IT 軌跡 |

> 行事曆對店長是「**入口型總覽**」，其餘 4 個是「**事件型細節**」。

### 3.4 「我的店」是怎麼判定的

底層查詢支援「指定門市」過濾，但**沒有「依登入者自動鎖店」邏輯**。店長視角需在後端額外加上：

- 取得登入者所屬門市
- 強制把門市條件覆寫為「自店」
- 不允許前端傳其他門市

> 待確認：多店店長（區內代理數家店）如何在行事曆切換店別。

## 4. 情境說明

### 4.1 正常流程 — 週日晚上排下週班

5/17（日）晚上，BK001 信義店店長小陳：

1. 打開「物流管理行事曆」，預設帶到本週
2. 切到「下一週」（5/18～5/24）
3. 看到：
   - 週一：週配（牛肉餅、麵包、生菜）
   - 週三：週配（牛肉餅補單）
   - 週五：月配（紙袋 1000 個、清潔用品）
4. 小陳把週一與週五排兩名點收同事，週三只留一名

### 4.2 行銷活動前確認到貨

5 月底有兒童節促銷，要送贈品玩具：

1. 切到 5/27（一）
2. 該日顯示「月配：兒童套餐玩具 200 套」
3. 確認 5/27 早上 9:00 前能備齊
4. 通知門市同仁該日加強人力

### 4.3 異常情境 — 看到「該到沒到」

5/19 早上 11:00，原本行事曆顯示今天應有週配，店長還沒收到：

1. 點 5/19 的日曆格
2. 看到本日週配明細（牛肉餅 12 公斤等）
3. 點某筆 → 跳「物流配送查詢」看車輛狀態
4. 若車輛已在門市附近卻沒到 → 聯絡司機；若已標延誤 → 進「物流異動查詢」確認

### 4.4 跨店店長情境（待確認）

某區經理代管 3 家店：

1. 行事曆上方有「店別」切換器
2. 預設自選的主要店
3. 切換後行事曆內容重新撈取

> 待確認：是否真有此需求，或區經理應走另一個總部視角功能。

### 4.5 公休／無配送日

某店週二公休、5/20 是公休日：

1. 5/20 那格顯示「公休」灰底
2. 不顯示任何配送
3. 點開不彈出明細

> 待確認：公休日資料來源（門市基本檔？BHM 模組？）

## 5. 操作流程

> 後端目前無店長視角 endpoint，以下為「應有的呼叫鏈」（推論）。

```
[店長打開「物流管理行事曆」]
  |
  +- 取得登入者身份 → 鎖定「自店」
  |
  +- 預設帶入本週（或本月）
  |    +- 撈該區間內、自店的「預計配送日」明細
  |    +- 後端依日期 + 配送模式分組
  |
  +- 行事曆畫面渲染：
  |    +- 每日格子顯示「週配 N 筆／月配 M 筆」
  |    +- 標記異常（延誤、短裝） — 由「物流異動查詢」回饋
  |
  +- 點某日 → 彈窗或側欄顯示當日明細
  |    +- 列：品項、需求量、來源單號、配送模式
  |    +- 行尾：跳「物流規劃查詢」「物流配送查詢」按鈕
  |
  +- 視圖切換：日／週／月
```

## 6. 欄位規格

### 6.1 行事曆主畫面（每日格子）

| 欄位（業務名） | 說明 | 店長視角是否該顯示 |
|---|---|---|
| 日期 | 行事曆 key | 是 |
| 該日配送筆數 | 各配送模式分別計數 | 是 |
| 配送模式分群 | 週配／月配／直送 | 是 |
| 異常標記 | 該日有延誤／短裝 | 是（推論，待補資料來源） |
| 公休標記 | 該店該日無營業 | 是（待確認資料來源） |

### 6.2 單日明細彈窗

| 欄位（業務名） | 說明 |
|---|---|
| 食材名稱 | 從食材主檔帶 |
| 需求量 | 來自需求預測明細 |
| 配送模式 | 週配／月配／直送 |
| 預計配送日 | 同當日 |
| 來源單號 | 需求預測單號或臨時需求單號 |
| 配送狀態 | 待派／已派／在途／已到（推論，需與配送查詢聯動） |

### 6.3 篩選條件

| 欄位（業務名） | 必填 | 說明 |
|---|---|---|
| 視圖類型 | 是 | 日／週／月 |
| 起始日期 | 是 | 預設今日 |
| 門市 | 系統自填 | 鎖定為登入者自店 |
| 配送模式 | 否 | 可只看週配或只看月配 |

## 7. 商業邏輯

### 7.1 自動鎖店

店長視角開啟行事曆時，後端應：
1. 從登入身份取得所屬門市
2. 把門市條件覆寫為「自店」，忽略前端任何門市參數
3. 若使用者非店長身份（例：總部誤入），則回到總部視角邏輯

### 7.2 行事曆預設視圖

- 預設「**月**」視圖，因排班與配送節奏多以月為單位
- 切換到「日／週」時保留鎖店條件
- 起始日預設今日

### 7.3 同日多配送模式分組規則

底層查詢已支援「日期 → 配送模式 → 明細」三層結構。店長視角直接沿用，前端在日曆格內以小區塊呈現各模式。

### 7.4 異常標記的來源（推論）

行事曆本身不記錄異常，異常資訊來自「物流異動查詢」累積的數據。建議：
- 每日定時或即時同步異動明細
- 將「該日有任一筆異動」反推為「日格顯示紅點」

> 此整合在後端尚未實作（待確認）。

### 7.5 唯讀

店長視角全程**只能查、不能改**。不開放新增、修改、刪除按鈕。

## 8. 使用角色與權限

| 角色 | 可看資料 | 可操作 |
|---|---|---|
| 店長（自店） | 自店預計配送日 | 唯讀、跳轉其他物流功能 |
| 副店長／代班 | 自店預計配送日 | 唯讀（推論） |
| 區經理（多店） | 區內各店切換 | 唯讀（待確認權限） |
| 總部物流規劃 | 全公司 | 走總部視角（非本功能） |

> 後端目前以「具備需求預測查詢權限」的角色控管，未細分店長／總部。店長視角的鎖店過濾需另實作。

## 10. 功能範圍

### 10.1 包含的功能

- 自店預計配送日行事曆（日／週／月視圖）
- 同日多配送模式分區呈現
- 單日明細彈窗
- 跳轉至兄弟物流功能

### 10.2 預留但尚未實作

| 項目 | 證據 |
|---|---|
| 店長視角專屬 endpoint | 後端無 `/app/*` 對應路徑 |
| 自店過濾（鎖店） | 既有 API 接受任意門市條件，無依登入者過濾 |
| 異常標記（紅點） | 後端無「該日異動彙總」查詢 |
| 公休日資料源 | 門市公休資訊未確認來源 |
| 行動裝置版面 | 後端僅提供資料，前端 RWD 未實作 |
| 推播提醒（早上 6:00 提示今日配送） | 後端排程未實作 |

### 10.3 不包含

- 修改預計配送日（屬需求預測模組）
- 即時車輛 GPS（屬「物流配送查詢」）
- 收貨點收（屬庫存入庫）
- 異動單建立（屬「物流異動查詢」）
