## 1. 功能概覽

### 1.1 我是誰

我是漢堡王台灣 **倉儲主管 / 區經理**。每月 / 每季要進行倉庫盤點，我建立「盤點計劃」決定：

> 「下個月第一週要盤點北一區 MSS 倉的『冷凍類』食材；第二週要盤『常溫類』；指派張三盤 A 區、李四盤 B 區」

盤點計劃 = 規劃要盤誰 / 何時 / 由誰執行；後續 #45 是執行端、#43 是盤點結果記錄。

### 1.2 我要做什麼

- 建立 / 編輯盤點計劃（單頭 + 品項清單）
- 走簽核流程
- 分頁查詢、待簽分頁、Excel 匯出
- 查詢某計劃的品類清單（透過 `check-plan-item` 子表）

### 1.3 我有什麼需求

| 我的需求 | 背後的問題 |
|---|---|
| 規劃要盤的範圍 | 全倉盤太累，分週分區進行 |
| 走簽核 | 盤點計劃需主管核可 |
| 規劃 → 執行 → 結果三層追溯 | 計劃編號可追溯到 #45 執行單與 #43 盤點結果 |

### 1.4 因此，需要以下功能

| 功能 | 解決的問題 |
|---|---|
| 盤點計劃 CRUD（單頭 + 品類） | 主要建單 |
| BPM 簽核 | 核可 |
| 與 #45 串接 | 簽核後計劃落到 #45 執行 |

### 1.5 功能基本資訊

| 項目 | 內容 |
|---|---|
| 功能中文名 | 盤點計劃製定管理 |
| 所屬模組 | WHS（庫存管理） |
| 兄弟功能 | 盤點作業 (#43)、計劃執行 (#45)、每日盤點 (#46) |
| 主要頁面 | 計劃編輯頁、單頭分頁、待簽分頁、Excel 匯出 |
| 簽核流程 | 有：`/todo-page` 端點存在 |
| 程式碼 Tag | 「盤點計劃制定」 |

---

## 2. 功能目的

盤點計劃是「**盤點活動的事前規劃**」：

1. **規劃端** — 決定範圍 / 時間 / 負責人
2. **承下 #45 執行端** — 簽核歸檔後計劃落到執行單
3. **完整管控** — 計劃 → 執行 → 結果三階段

---

## 3. 業務邏輯背景

### 3.1 兩張表

| 表 | 用途 |
|---|---|
| `whs_check_plan`（單頭 / `CheckPlanDO`） | 單據編號、流程狀態、區域、倉別、時間區間等 |
| `whs_check_plan_item`（品項清單 / `CheckPlanItemDO`） | 計劃要盤的品類 / 品號 |

### 3.2 與 #45 / #43 的關係

- #44 → 規劃要盤什麼
- #45 → 把規劃落到具體執行單據（CheckPlanDetail）
- #43 → 實際盤點當天的記錄（CheckTake）

三者連動，分立避免一張單既當規劃又當執行。

### 3.3 跨模組依賴

- 與 #45 串接（透過 signCode 或 ID）
- BPM 簽核

```

  │
  ├─ POST /whs/check-plan/create
```
