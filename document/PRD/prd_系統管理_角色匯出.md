## 1. 功能概覽

### 1.1 我是誰

我是漢堡王的稽核／資安窗口（或系統管理員）。
公司各部門員工在 ERP 中分屬不同角色（採購主管、門市店長、倉儲管理員、財會等），角色控制著「誰能看什麼、能做什麼」。我需要定期把目前系統內所有角色的設定狀況交給稽核單位查核，或是在權限改動前留下「改之前長什麼樣」的書面證據。

### 1.2 我要做什麼

- **每季**：把目前 ERP 內的全部角色匯出 Excel，附在稽核資料包裡交給稽核單位。
- **每月**：權限政策有調整時，先匯出一份「異動前」的角色清冊作為對照基準。
- **不定期**：法遵、資安、外部稽核臨時索取「目前系統有哪些角色、各自的資料可視範圍是什麼」的清冊。
- **不定期**：跨單位主管要對齊權限分工時，匯出 Excel 在會議上做為討論底稿。

### 1.3 我有什麼需求

| 我的需求 | 背後的問題 |
|---|---|
| 一次拿到系統內所有角色的完整清單 | 稽核要求書面證據，不能逐筆截圖 |
| 能依條件篩選（如只取啟用中的） | 已停用的角色未必有意義列入稽核基準 |
| 結果要含「數據範圍」資訊 | 稽核關注「這個角色到底能看到多少資料」 |
| 匯出動作要留下「誰、何時匯出」的紀錄 | 角色清冊含敏感資訊，事後要可追溯誰拿過 |
| 不需要每次都動 IT，自己有權限就能匯出 | 稽核時程急，等 IT 排班會誤期 |

### 1.4 因此，需要以下功能

| 功能 | 解決的問題 |
|---|---|
| 以條件篩選後一次性匯出角色清單 Excel | 取得稽核／對齊用的書面證據 |
| 篩選欄位涵蓋角色名稱、角色標識、狀態、建立時間區間 | 支援不同稽核情境（依名稱搜尋、依時間區段、依狀態） |
| 自動套用本人的權限範圍 | 不是所有人都能匯出，只有具備「系統-角色-匯出」權限者可執行 |
| 匯出動作寫入存取日誌 | 事後可追溯「誰在何時匯出角色清冊」 |

### 1.5 功能基本資訊

| 項目 | 內容 |
|---|---|
| 功能中文名 | 角色匯出（Excel） |
| 所屬模組 | 系統管理 |
| 兄弟功能 | 角色新增／修改／刪除、角色分頁查詢、角色精簡清單（給下拉用）、角色資料範圍指派、角色—選單授權 |
| 主要頁面 | 角色管理列表頁的「匯出」按鈕 |
| 簽核流程 | 無 |

## 2. 功能目的

把「目前系統內所有角色（含其名稱、標識、排序、狀態、資料可視範圍）」一次性輸出為 Excel 檔，作為稽核佐證、權限變更前的快照、或跨單位對齊權限分工的文件底稿。本功能本身**不會修改任何資料**，僅是把資料庫現況以欄位精選後的 Excel 形式輸出，並在伺服端寫入一筆「匯出」存取日誌。

> 經系統整合決策（2026-05-16），漢堡王 IT 走向以 Laravel OA 為使用者單一事實來源、ERP 不再自簽 JWT。**「角色」概念長期將由 OA 統一管理**，本模組屬於 ERP 過渡期保留功能；長期規劃中將在搬遷階段 5 後廢棄。本 PRD 描述的是過渡期內、ERP 仍自管角色主檔時的需求。

## 3. 業務邏輯背景

### 3.1 角色分兩類：系統內建 vs 自訂

ERP 內的角色有兩種類型：

- **系統內建角色**（如超級管理員）— 由系統初始化時建立，禁止修改與刪除
- **自訂角色**（IT 後續新增）— 可以正常增刪改

匯出功能**不會區分這兩類**，所有角色都會匯出。但稽核時這個分類很重要，§11 列為待確認是否需在匯出檔加上「角色類型」欄位。

### 3.2 「數據範圍」決定一個角色能看到哪些業務資料

每個角色都掛一個「數據範圍」設定，這是 ERP 資料權限的核心：

| 數據範圍 | 意思 |
|---|---|
| 全部資料 | 不設限，可看到系統內所有資料（常用於管理員） |
| 指定部門資料 | 僅可看到特定幾個部門（IT 另外指派部門清單） |
| 本部門資料 | 僅可看到自己所屬的那一個部門 |
| 本部門及以下資料 | 自己所屬部門及其下層部門 |
| 僅本人資料 | 只能看自己經手的單據 |

匯出檔中會用文字方式呈現此欄位（透過字典轉換），讓稽核可以直接讀懂。

### 3.3 角色名稱與角色標識各自全系統唯一

- **角色名稱**（例：「採購主管」）— 顯示用
- **角色標識**（例：`procurement_manager`）— 程式內判斷用、不可重複

匯出時兩欄都會出現。

### 3.4 匯出採「全量、不分頁」模式

平常的查詢頁是分頁的（一頁 N 筆）。匯出時系統強制把分頁參數設為「不分頁」，把所有符合條件的角色一次取出寫入 Excel。

### 3.5 匯出採用伺服端字典轉換

「狀態」「數據範圍」這類用數字代碼儲存的欄位，匯出時會在伺服端就轉成中文字（「啟用」「停用」「全部資料」⋯），使用者開啟 Excel 不需再查代碼表。

## 4. 情境說明

### 4.1 正常流程 — 季度稽核匯出

5/18 下午稽核員小郭向 IT 索取「目前系統內全部角色清單」。IT 阿明開啟角色管理列表頁，不設任何篩選條件，直接點「匯出」。瀏覽器下載一支名為 `角色数据.xls` 的檔案，內含 23 個角色（含 1 個內建超級管理員、22 個自訂角色），每筆有：角色序號、角色名稱、角色標識、角色排序、角色狀態、數據範圍 6 個欄位。阿明把檔案上傳到稽核共享資料夾。同時系統留下一筆「2026/05/18 14:32 由 amin 執行匯出」的存取日誌。

### 4.2 條件匯出 — 只取啟用中的角色

5/20 法遵單位要做「目前活躍角色」盤點。阿明在篩選列填入「狀態：啟用」後按匯出，得到 19 筆（停用中的 4 筆不在內）。

### 4.3 條件匯出 — 依名稱模糊找

阿明要看所有名稱含「採購」的角色設定。在角色名稱欄位輸入「採購」按匯出，得到 5 筆（採購主管、採購助理、總部採購、門市採購、採購代理人）。

### 4.4 條件匯出 — 依建立時間區間

權限政策在 4 月做了一次大盤點，阿明要對照「4 月後新增的所有角色」。設定建立時間：`2026-04-01 00:00:00` 至 `2026-04-30 23:59:59`，匯出得到 3 筆。

### 4.5 異常情境 — 無匯出權限

新進採購主管小陳登入系統，看到角色管理頁但「匯出」按鈕灰階／看不到，因為他的角色未配發「系統-角色-匯出」權限。即使他直接呼叫匯出網址，伺服端會拒絕。

### 4.6 邊界情境 — 篩選後無資料

阿明輸入一個不存在的角色名稱「XXXYYY」按匯出，仍會下載一個 Excel，但內容只有表頭、無資料列。

### 4.7 自動記錄情境 — 存取日誌

阿明每次匯出，系統都自動在「API 存取日誌」中新增一筆，操作類型為「匯出」。稽核日後可在存取日誌頁查到「2026 年內共有誰、在哪天、匯出過幾次角色清冊」。

```
[使用者進入角色管理列表頁]
  │
  ▼
[填入篩選條件（可全空）]

  ├─ 角色標識（模糊）
  ├─ 角色狀態（啟用 / 停用）
  └─ 建立時間區間
  │
  ▼

  │
  ▼

  │
  ▼
[將分頁參數覆寫為「不分頁」]
  │
  ▼
[依篩選條件查詢所有符合的角色]
  │
  ▼

  │
  ▼
```

| 不出現欄位 | 為何稽核可能在意 |

## 7. 商業邏輯

### 7.1 匯出範圍 = 篩選後的「全量」

呼叫匯出時，系統會把分頁大小強制設為「不分頁」，把所有符合篩選條件的角色一次取出。**無筆數上限**，理論上若系統內有 5 萬個角色，會一次拉出 5 萬筆。實務上 ERP 角色數量極少（推論：通常 < 100），此設計無風險，但若未來角色數激增，匯出可能變慢。（來源：`RoleController.java:93`）

### 7.2 篩選條件組合

匯出採用與分頁查詢相同的篩選邏輯：

- 角色名稱、角色標識：**模糊**比對（部分相符即取）
- 角色狀態：**等於**比對
- 建立時間：**區間**比對（起訖含界）

所有條件以 AND 串接。任一條件未填則該條件不參與篩選。

### 7.3 排序

以角色的「排序」欄位升冪排列輸出（與列表頁顯示順序一致）。

### 7.4 不過濾系統內建角色

匯出**不會**自動排除「系統內建」類型的角色。所有符合篩選的角色皆會列入，含超級管理員。稽核情境下這通常是預期行為（內建角色是稽核重點），但若不同情境想排除則無此選項。

### 7.5 不過濾資料權限

一般查詢可能會受「該登入者只能看本部門資料」之類的資料權限限制。本匯出端點未明確套用資料權限攔截（推論：因角色屬全系統共用資料，無部門歸屬概念，故資料權限框架不會作用於此資料表），§11 列為待確認。

### 7.6 字典轉換在伺服端執行

「角色狀態」「數據範圍」兩欄位以代碼儲存，匯出時透過字典轉換器把代碼換成可讀文字寫入 Excel，使用者直接看 Excel 即可，不需要再去查代碼表。

### 7.7 存取日誌

呼叫匯出端點時自動寫入一筆 API 存取日誌，操作類型標記為「匯出」。日誌內含：執行者、時間、來源 IP、執行耗時、回應狀態。**目前不會記錄使用者實際下了什麼篩選條件**（推論），§11 列為待確認。

### 7.8 輸出檔案

- 檔名：`角色数据.xls`
- 工作表名：`数据`
- 格式：Excel 97-2003（`.xls`）

## 8. 使用角色與權限

| 角色 | 可看／可操作 |
|---|---|
| **具備「系統-角色-匯出」權限者**（通常為系統管理員、稽核窗口） | 可執行匯出 Excel |
| **僅具備「系統-角色-查詢」者** | 可在頁面看到角色列表，但無匯出按鈕／伺服端會拒絕匯出請求 |
| **未配發角色管理權限者** | 看不到角色管理選單 |

匯出權限與查詢權限**獨立掛載**，可分別配發。例如可以「讓某稽核窗口能匯出但不能新增／改／刪」，或反之。

## 10. 功能範圍

### 10.1 包含的功能

- 以條件篩選（角色名稱、角色標識、狀態、建立時間區間）匯出角色清單為 Excel
- 匯出檔欄位含：角色序號、角色名稱、角色標識、角色排序、角色狀態、數據範圍
- 自動寫入「匯出」類型的存取日誌
- 採權限隔離（須具備「系統-角色-匯出」權限）

### 10.2 預留但尚未實作

- 「角色類型（內建／自訂）」、「指定部門清單」、「備註」、「建立時間」四個欄位在角色資料中存在，但未被納入 Excel 輸出。
- 篩選條件中未提供「角色類型」篩選（雖然底層資料有此欄位）。

### 10.3 不包含

- 匯出後的歸檔／自動上傳到稽核共享儲存
- 排程式定期匯出（目前僅支援使用者手動點擊觸發）
- 匯出檔的數位簽章／浮水印
- 「使用者—角色」對應關係的匯出（屬於使用者管理，不在本功能）
- 「角色—選單授權」對應的匯出（屬於權限分配，不在本功能）
