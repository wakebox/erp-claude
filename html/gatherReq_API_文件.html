① 需求試算 / 需求預測排程 (Demand Forecast)

前端路由
/gatherReq/reqCalculation/detail ｜ /gatherReq/demandForecastSchedule/detail
Controller: DemandForecastController / DemandForecastDetailControllerService: DemandForecastServiceImpl表: crg_demand_forecast (+_detail)

功能說明

依「歷史銷售資料區間」與「需求週別」，把門市銷售量按 每萬元營業額 換算成平日 / 假日的平均食材需求量，再乘上配方標準用量與預測加成 (%)，產生一張可送簽核的「需求預測試算表」。表頭含週別、銷售資料區間、預測加成；明細按 門市→商品→食材 三層展開。

核心計算公式

avgSales = sales / dayCount （平日或假日：總銷量 ÷ 天數） avgSalesPer10k = avgSales × 10000 / orderAmount （每萬元營業額平均銷量） projectedPer10k = avgSalesPer10k × standardQuantity （乘配方標準數量 → 食材需求） 最終預測 = projectedPer10k × forecastIncrementPercent （再乘預測加成%） 零值保護：orderAmount ≤ 0、dayCount ≤ 0、standardQuantity ≤ 0 → 回傳 0 四捨五入：中間運算 8 位、輸出 2 位 (HALF_UP) LONG(常溫)食材：庫存未低於安全庫存則跳過試算，前端顯示「判斷庫存」

每萬元平均需求量 (DemandForecastServiceImpl:631-649)

private BigDecimal calculateAverageSalesPer10k(Integer sales, BigDecimal orderAmount, Integer dayCount) {
    BigDecimal validOrderAmount = orderAmount == null ? BigDecimal.ZERO : orderAmount;
    BigDecimal validSales       = sales == null ? BigDecimal.ZERO : BigDecimal.valueOf(sales);
    BigDecimal validDayCount    = dayCount == null ? BigDecimal.ZERO : BigDecimal.valueOf(dayCount);

    if (validOrderAmount.compareTo(BigDecimal.ZERO) <= 0
            || validDayCount.compareTo(BigDecimal.ZERO) <= 0) {
        return BigDecimal.ZERO;
    }
    return validSales
            .divide(validDayCount, 8, RoundingMode.HALF_UP)   // 平均每日
            .multiply(new BigDecimal("10000"))                // ×萬元
            .divide(validOrderAmount, 8, RoundingMode.HALF_UP);
}

② 臨時需求試算 (Temporary Requirement)

前端路由
/gatherReq/temporaryCalculation/detail

功能說明

門市臨時加單時，選擇單品 (recipe) 並輸入「申請臨時需求量 appliTempNum」，系統依單品配方 (pdm_single_serving_recipe) 的每份標準用量，即時試算每項食材應備數量。與需求預測不同，臨時需求是直接量×配方的簡單展開，不涉及銷售統計。

核心計算公式

tempReqQuantity = standardQuantity × appliTempNum （標準數量 × 申請臨時需求量，四捨五入2位） tempReqFinalNum = tempReqUnit (若 > 0) 否則 tempReqQuantity （有手動覆寫值則優先）

臨時需求試算 (TempReqServiceImpl:164-189)

// 申請臨時需求數量
Integer appliTempNum = tempReqDetailDO.getAppliTempNum();
for (TempReqDetailDTO dto : tempReqDetailDTOList) {
    BigDecimal standardQuantity = dto.getStandardQuantity();  // 標準數量
    if (standardQuantity != null && standardQuantity.compareTo(BigDecimal.ZERO) > 0) {
        BigDecimal tempReqQuantity = standardQuantity
                .multiply(new BigDecimal(appliTempNum))
                .setScale(2, BigDecimal.ROUND_HALF_UP);       // 臨時需求計數
        dto.setTempReqQuantity(tempReqQuantity);
    }
    // 臨時需求最終值：有手填單位量優先，否則用計數
    BigDecimal finalNum = (dto.getTempReqUnit() != null && dto.getTempReqUnit().compareTo(BigDecimal.ZERO) > 0)
            ? dto.getTempReqUnit() : dto.getTempReqQuantity();
    dto.setTempReqFinalNum(finalNum);
}

週配物流日計算 (RawMaterialDemandHeadServiceImpl:532-556)

private static LocalDateTime calculatePreviousWeekLogisticsDate(LocalDateTime targetDate, String logisticsCycle) {
    List daysOfWeek = Arrays.stream(logisticsCycle.split(","))
        .map(String::trim).map(Integer::parseInt).map(DayOfWeek::of)
        .sorted(Comparator.comparingInt(DayOfWeek::getValue).reversed())
        .collect(Collectors.toList());
    LocalDateTime currentDate = targetDate;
    for (int i = 1; i <= 7; i++) {
        LocalDateTime candidate = currentDate.minusDays(i);
        if (daysOfWeek.contains(candidate.getDayOfWeek())) {
            return candidate.with(LocalTime.MIN);
        }
    }
    throw new IllegalStateException("No previous logistics date found for " + logisticsCycle);
}

分組 SQL (RawMaterialDemandDetailMapper.xml:177-206)

SELECT rmdd.region_id AS regionId,
       rmdd.expect_delivery_date AS expectDeliveryDate,
       MIN(rmdd.demand_store) AS demandStore,
       MIN(rmdd.store_id)     AS storeId,
       STRING_AGG(DISTINCT rmdd.demand_relation_doc, ',') AS demandRelationDoc,
       STRING_AGG(DISTINCT rmdd.temp_relation_doc, ',')   AS tempRelationDoc
FROM pdm_raw_material_demand_detail rmdd
WHERE rmdd.deleted = 0 AND rmdd.region_id = #{regionId}
   AND rmdd.store_id = #{storeId} 
  AND rmdd.expect_delivery_date IS NOT NULL
   AND rmdd.expect_delivery_date >= #{startDate} 
     AND rmdd.expect_delivery_date <= #{endDate}   
GROUP BY rmdd.region_id, rmdd.expect_delivery_date
ORDER BY rmdd.expect_delivery_date

實際到店量比例分攤 (StockRecordHeadServiceImpl:247-291)

for (int i = 0; i < relatedDetails.size(); i++) {
    RawMaterialDemandDetailDO d = relatedDetails.get(i);
    BigDecimal allocated;
    if (i == relatedDetails.size() - 1) {                 // 最後一筆用餘額避免精度誤差
        allocated = totalActualAmount.subtract(allocatedSum).setScale(2, BigDecimal.ROUND_HALF_UP);
    } else if (totalDemandAmount.compareTo(BigDecimal.ZERO) > 0) {
        allocated = d.getDemandAmount()
            .divide(totalDemandAmount, 2, BigDecimal.ROUND_HALF_UP)   // 需求佔比
            .multiply(totalActualAmount).setScale(2, BigDecimal.ROUND_HALF_UP);
        allocatedSum = allocatedSum.add(allocated);
    } else { allocated = BigDecimal.ZERO.setScale(2); }
    d.setActualArrivalAmount(allocated);
    d.setActualArrivalDate(actualArrivalDate);
    d.setStockSignCode(stockSignCode);                    // 回填入庫單號
    rawMaterialDemandDetailMapper.updateById(d);
}
