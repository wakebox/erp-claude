採購流程總覽

PMM（Procurement / Material Management，採購物料管理）涵蓋「採購主鏈」與兩張側支主檔。完整資料流如下：

商業邏輯

  • create：generateSignCode("請購單管理") 產編號；processStatus = PENDING_PROCESS；明細批次插入；若簽核開啟則啟動 PURCHASE_REQUISITION 流程。
  • update（狀態觸發）：已歸檔且無流程不可改；明細採「先全刪後重建」。若本次 processStatus="已歸檔" 且舊值非已歸檔（首次歸檔）→ 呼叫 createQuoteFromPurReq(id) 自動生成報價單(PBP)。
  • createQuoteFromPurReq：以 reqSignCode 做冪等保護（一張請購只生成一張報價）；逐筆轉 QuoteDetailDO（遞增 reqItem、帶來源單號）並啟 QUOTATION 流程。
  • 請購量試算公式：standardQuantity = (safeStock − 目前庫存量) × weightFactor（三值皆非 null 才計算）。

商業邏輯

  • update 校驗：歸檔時逐筆檢查 defaultSupplier（選定廠商）不可為空，否則拋 QUOTE_DEFAULT_NOT_EXISTS。歸檔時單頭/明細 status="2"（採購中）。
  • generateNewPurOrder（核心：轉採購 + 金額計算）：依 mfrId 分組，每廠商各開一張採購單。逐筆計算：
    • 採購數量 purQty = purQuantity ÷ singlePackCount（ROUND_UP，不足一箱進一箱）
    • 行金額 lineAmount = purQty × unitPrice（每箱價，scale 3 HALF_UP）
    • 稅額依 taxType："0"（營業稅）→ totalAmount = untaxedAmount × 1.05；其餘 → totalAmount = untaxedAmount；taxAmount = totalAmount − untaxedAmount
    • 每張採購單啟 PURCHASING 流程
