建立流程實例（Submit 時）

業務單據提交 → 觸發簽核流程
ℹ️ 每個業務 Service（食材、單品、採購單…）呼叫 MenuFlowProcessInstanceHelper.createProcessInstanceIfFlowOpen()，若 flowIsOpen=0 則建立 Flowable 流程實例，回傳的 ID 存入業務單據。
❌ 現況（Flowable）
// MenuFlowProcessInstanceHelper.java:45
public Optional<String> createProcessInstanceIfFlowOpen(
    Long userId, String menuFlowPath, Long businessId) {

  MenuDO menu = menuService
    .getMenuByFlowpath(menuFlowPath);

  if (menu == null || !isOpen(menu)) {
    return Optional.empty();
  }

  String businessKey =
    menu.getPath() + ":" + businessId;

  String instanceId =
    bpmProcessInstanceApi
      .createProcessInstance(userId,
        new BpmProcessInstanceCreateReqDTO()
          .setProcessDefinitionKey(
              menu.getFlowKey())
          .setBusinessKey(businessKey));

  return Optional.of(instanceId);
}
✅ 替換後（OA API）
// OaWorkFlowHelper.java（新增）
public Optional<String> createProcessInstanceIfFlowOpen(
    Long userId, String menuFlowPath, Long businessId) {

  MenuDO menu = menuService
    .getMenuByFlowpath(menuFlowPath);

  if (menu == null || !isOpen(menu)) {
    return Optional.empty();
  }

  String businessKey =
    menu.getPath() + ":" + businessId;

  String workFlowId =
    oaWorkFlowApiClient
      .createWorkFlow(
        new OaWorkFlowCreateDTO()
          .setTemplateId(
              menu.getFlowKey()) // ← 改為 OA template_id
          .setBusinessKey(businessKey)
          .setUserId(userId));

  return Optional.of(workFlowId);
}
Flowable 欄位OA 欄位
processDefinitionKey→template_id（菜單 flowKey 改存此值）
businessKey（"path:id"）→content JSON 內的 businessKey 欄位
userId（Long）→user（OA 使用者 ID，需對應）
回傳 processInstanceId（String）→回傳 wf_work_flows.id（業務單據存此值）
2

流程完成回調（最關鍵整合點）

簽核通過 / 拒絕 → 業務單據狀態更新
⚠️ 現況依賴 Flowable 內部 Spring Event 機制；OA 是 PHP 系統，無法直接發 Spring Event。必須在兩端各加一個接縫（OA 的 Trigger + BK ERP 的 Webhook）。
🔔 做法 A：OA 新增 Trigger 呼叫 BK ERP Webhook 推薦
OA 完成流程時，透過現有 Strategy 模式呼叫 BK ERP 暴露的 callback endpoint。
✅即時；架構乾淨；符合 OA 現有 Strategy 設計
⚡需改動 OA 程式（新增一個 Trigger class）
🔄 做法 B：BK ERP 輪詢 OA 狀態
Spring Scheduler 定時呼叫 OA 的 GET /api/wf/work-flow/processing，比對 processInstanceId（即 OA wf id）的狀態。
✅OA 零改動
⚡不即時（秒級延遲）；增加 OA 查詢壓力
做法 A 實作細節
PHP app/Strategy/WF/ToBkErpTrigger.php（OA 新增） 
class ToBkErpTrigger implements WorkFlowStrategyInterface {

    public function execute(WorkFlow $workFlow): void {
        $content = json_decode($workFlow->content, true);
        $businessKey = $content['businessKey'] ?? null;

        if (!$businessKey) return;

        Http::withToken(config('services.bk_erp.token'))
            ->post(config('services.bk_erp.callback_url'), [
                'businessKey' => $businessKey,
                'status'      => $workFlow->status, // C=通過, R=拒絕
                'workFlowId'  => $workFlow->id,
            ]);
    }

    public function supports(WorkFlow $workFlow): bool {
        return $workFlow->model_type === 'bk';
    }
}
Java OaCallbackController.java（BK ERP 新增） 
@RestController
@RequestMapping("/bk/callback")
public class OaCallbackController {

    @Resource private ApplicationEventPublisher eventPublisher;

    @PostMapping("/wf-complete")
    public CommonResult<Void> onWorkFlowComplete(
            @RequestBody OaCallbackReqDTO req) {

        // businessKey 格式："menuPath:businessId"（與原 Flowable 相同）
        String[] parts = req.getBusinessKey().split(":", 2);
        int status = "C".equals(req.getStatus())
            ? BpmTaskStatusEnum.APPROVE.getStatus()
            : BpmTaskStatusEnum.REJECT.getStatus();

        eventPublisher.publishEvent(
            new BpmProcessInstanceStatusEvent(this)
                .setBusinessKey(req.getBusinessKey())
                .setStatus(status));

        return success(null);
    }
}
💡 重用現有 BpmProcessInstanceStatusEvent：各模組的 *StatusListener 不用改一行。BK ERP 側唯一新增的是這個 webhook controller，和 OA 側的一個 Trigger class。

查詢類型Flowable 方法OA 替換 API篩選邏輯
待辦 .unfinished().taskAssignee() GET /api/wf/work-flow/handling content.businessKey 前綴匹配

⚠️ OA 的 content 是 JSON 字串，businessKey 需存在固定的 JSON key 內（如 content.businessKey）。建立流程時要確保格式一致，否則這邊的篩選會失效。

✅ 替換後
// flowOptions.ts:8
export async function getFlowOptions(
    type: string) {

  // 改查 OA template steps
  const steps = await getOaTemplateSteps(type);
  // GET /api/wf/template-flows?template_code=type

  steps.forEach(step =>
    list.push({
      value: step.name,
      label: step.name,
      sequence: step.sequence
    })
  );
}
💡 OA 的 wf_template_flows 有 sequence 和簽核人名稱，可直接對應 Flowable 的 BPMN task node。只需在 OA 側暴露一個「取得 template 步驟清單」的 API（若尚未有）。
flowKey 欄位意義變更
▸ MenuDO.flowKey 的角色轉換
情境flowKey 的值
現況（Flowable） → Flowable processDefinitionKey，例如 "ingredients"、"prm"
